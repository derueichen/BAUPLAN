# BAUPLAN · 任務建構系統

Bauplan 是德文「建築藍圖」。把大目標拆成一天做得完的小任務，每天推進一格——原子習慣式的專案管理工具。

單一 `index.html`，零框架、零建置流程、零後端，直接放上 GitHub Pages 就能用。

## 功能

- **今日**：早晨儀表板。今天的任務清單、完成率圓環、逾期提醒（可一鍵「移到今天」）、連續完成天數、明日與未來 7 天預告。
- **專案**：flight deck 式總覽——所有專案以磚塊排在同一畫面，一眼看到每個目標的剩餘量、今日與逾期數、下一步任務。按住 ⠿ 拖曳磚塊即可調整優先順序（今日清單與甘特圖的排序會跟著改）；點磚塊在下方展開該專案的任務明細。新增任務時可設定「重複」（每天／每週／每月 × 次數），一次產生自動編號的小任務，把一件大事拆散到很多天。
- **甘特圖**：雙月時間軸，左右換月。實心色塊＝已完成、斜紋＝進行中、紅線＝今天。直接點色塊即可切換完成狀態。
- **從 Markdown 建立**：右上「Markdown」按鈕。貼上一段有日期、有標題的 markdown，系統解析成待辦清單，先在預覽表逐列確認——可指定每列歸到哪個現有專案或新建、移除不要的列——按確認才真正寫入。已存在（同專案＋同日＋同名）的任務會自動略過，不會重複。

  慣例寫法：

  ```markdown
  # 地頭蛇計畫          ← 一級標題 = 專案名
  ## 2026-08-05         ← ## 或裸日期行 = 該段落的日期
  - 回覆協作單位聯繫    ← 任務，繼承段落日期
  - 教材更新 8/6 ~ 8/9  ← 行內寫日期區間 = 跨天任務
  - [x] 器材盤點        ← [x] 視為已完成
  ```

  日期支援 `2026-08-05`、`8/5`、`8月5日` 等寫法；只寫月日時自動補上今年。找不到日期的任務會暫定今日並在預覽時提示。

## 部署到 GitHub Pages

1. 在 GitHub 建立一個新的 repository（例如 `bauplan`），設為 Public。
2. 上傳 `index.html` 到 repository 根目錄（網頁介面按 **Add file → Upload files** 即可）。
3. 進入 **Settings → Pages**，Source 選 **Deploy from a branch**，Branch 選 `main`、資料夾選 `/ (root)`，按 Save。
4. 等一兩分鐘，網址會是 `https://<你的帳號>.github.io/bauplan/`。加到手機主畫面，每天早上打開就是今日儀表板。

## 資料與備份

- 所有資料存在**該瀏覽器的 localStorage**，不會上傳到任何伺服器。
- 換裝置、換瀏覽器或清除瀏覽資料前，請先按右上角「**匯出**」下載 JSON 備份，再到新環境「**匯入**」。
- 建議養成每週匯出一次的習慣（這也是一個可以排進 BAUPLAN 的重複任務）。

## 自訂

顏色與尺寸都集中在 `index.html` 開頭的 `:root` 變數：

```css
:root{
  --paper:#F6F4EE;  /* 底色 */
  --ink:#16130E;    /* 墨黑 */
  --red:#DF3418;    /* 包浩斯紅 */
  --blue:#1D57A5;   /* 包浩斯藍 */
  --yellow:#F0B41C; /* 包浩斯黃 */
  --dw:26px;        /* 甘特圖每日欄寬 */
}
```

自由使用、修改、分享。
